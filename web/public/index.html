CPSL

Safe, UNIX-like mini-OS capsules for agents, designed to run across Linux, macOS, Windows, web browsers, iOS, and Android.

Package tools, files, and permissions into a capsule. Build it. Run it.

Run capsules through Bash-compatible commands, Python-style scripts, or Luau code. 👇

cpsl-web capsule

Built with cpsl-web.toml Allowed domains httpbin.org

Describe, build, run.

A capsule starts as TOML. Give it a name, enable only the modules it needs, and allow specific network domains. Then use the CPSL CLI to build and run.

Build

$ cpsl build -f cpsl-web.toml
built cpsl-web

List

$ cpsl ls
NAME           MODULES         SIZE    BUILT
cpsl-web       fs, http, json  7.4 MB  2026-05-17

Run

$ cpsl run cpsl-web -- \
  'json decode "{\"hello\":\"world\"}"'
{
  "hello": "world"
}

How does it work?

CPSL runs a Luau VM and exposes selected Rust-backed modules inside each capsule.

⚙️ Luau VM

Luau is a small, fast, and embeddable programming language based on Lua with a gradual type system. It was built and open-sourced by Roblox and is battle-tested by millions of users.

🧩 Composable

Filesystem, networking, JSON, compression, custom modules...
If you only need JSON and HTTP for one domain, keep the manifest that small.

🎙️ Communication

Agents and humans can interact with CPSL using Bash, Python, or Lua/Luau.
A Luau runtime runs under the hood; Bash and Python are transpiled.

💡 Python-on-Luau

Python code can run faster when transpiled into Luau, even including transpilation time. 😅
See bench-python-luau.sh in the repo.

What CPSL is not

Not Linux

It looks and feels like Unix: programs, everything is a file, the FS tree is rooted at /, and there is a sh/bash-compatible shell, etc. What's important is that it's UNIX-like enough for agents.

Not Docker/OCI

No daemon. Not a Linux distribution. Not a container image.

Not CPython

Though CPSL can run Python code, it does not carry the actual Python tooling. No pip install.
