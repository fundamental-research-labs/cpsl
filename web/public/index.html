CPSL

Safe mini-OS "capsules" for agents that can run everywhere: Linux, macOS, Windows, web browsers, iOS, Android...)

Package tools, files, and permissions. Build. Run.

Agents can communicate with CPSL using Bash, Python, or Lua,
and so can you. 👇

cpsl-web capsule

Built with cpsl-web.toml Allowed domains httpbin.org

Describe, build, run.

A capsule starts as TOML. Name your sandbox, pick modules, and pin network domains. Then use the CPSL CLI to build and run. File mounts and allowed domains can be edited while the capsule is running.

Build

$ cpsl build -t browser-agent -f cpsl-web.toml
built browser-agent

List

$ cpsl sandboxes
NAME           MODULES         SIZE    BUILT
browser-agent  fs, http, json  7.4 MB  2026-05-06

Run

$ cpsl run browser-agent --python -- 'print("hello from inside")'
hello from inside

How does it work?

CPSL is a Luau VM that exposes Rust crate assemblies.

⚙️ Luau VM

Luau is a small, fast, and embeddable programming language based on Lua with a gradual type system. It was built and open-sourced by Roblox and is battle-tested by millions of users.

🧩 Composable

File system, networking, JSON, compression, custom modules...
If you just need JSON and HTTP with one allowed domain, then stick to the bare minimum.

🎙️ Communication

Agents and humans can interact with CPSL using Bash, Python, or Lua/Luau.
A Luau runtime runs under the hood; Bash and Python are transpiled.

💡 Python-on-Luau

Python code can run faster when transpiled into Luau, even including transpilation time. 😅
See ./run_tests.sh in the repo.

./run_tests.sh

What CPSL is not

Not Linux

It looks and feels like Unix: programs, everything is a file, the FS tree is rooted at /, and there is a sh/bash-compatible shell, etc. What's important is that it's Unix-like enough for agents.

Not Docker/OCI

No daemon. Not a Linux distribution. Not a container image.

Not CPython

Though CPSL can run Python code, it does not carry the actual Python tooling. No pip install.

Build from source

git clone https://github.com/fundamental-research-labs/cpsl
cd cpsl
./build-cli.sh
./cpsl -i

Early and hackable, come contribute!

CPSL was open-sourced on May 4, 2026. It's already used in production in some Fundamental Research Labs products. It's an extremely powerful piece of tech, but it is not yet perfectly modular; not all build targets are clearly exposed, etc. It's the right time to join as a contributor and help us design the perfect isolated, versatile operating system for AI agents.
